# Login and Tokens

When using our API to login with OAuth the first step is to add an onClick attribute to our JSX <button> on the Login page. We need to pass a redirect url through the querystring to your API endpoint. In the code below you can see how to navigate to the server-side url. The navigate() method that comes from the {useNavigate} hook only works for internal URLs, not external ones.

location.href = `https://your.serverside.api/auth/google?redirect_url=https://127.0.0.1:5173`;
1

Whatever URL you pass through the querystring will be the page that loads after you have successfully logged in.

When that URL loads, if it was successful then it will have a querystring parameter called token that includes your JWT string.

You can use the React Router {useSearchParams} hook to read for that parameter and {useNavigate} hook to navigate to the people list page.

const [searchParams] = useSearchParams();

useEffect(() => {
  if (searchParams.get('token')) {
    //there is a token in the querystring that can be
    //saved in sessionStorage
    navigate('/people');
    //then go to the people list page
  } else {
    //no token in the querystring
    //check if the token is in sessionStorage
    //if not then send the user back to the login page
  }
}, []);

# Context for Global Tokens

A Context Object is a great way to store and access the JWT token. The Context object will be able to read and store the token string. If it exists then the user is logged in. If it doesn't exist or is null then the user is not logged in.

You should create a hook for saving values in and reading values from sessionStorage. This Context object will use that hook to access the value of the token or to save that token in sessionStorage.

The Context creates and exports:

  1. The context provider component.
  2. The hook for accessing the context object or updating its value.

Here is an example of a Context object script.

import { createContext, useContext } from 'react';
import { useSessionStorage } from '../hooks/sessionStorage';

const TokenContext = createContext(); //create the context object
const key = 'myGiftrAppToken';

function TokenProvider(props) {
  const [token, setToken] = useSessionStorage(key, null);
  return <TokenContext.Provider value={[token, setToken]} {...props} />;
}

function useToken() {
  const context = useContext(TokenContext);
  if (!context) throw new Error('Not inside the Provider');
  return context; // [token, setToken]
}

//export the hook and the provider
export { useToken, TokenProvider };

The sessionStorage hook will be a function that accepts the name of the key to use for creating and accessing the sessionStorage value, plus an initial value to save.

The value from sessionStorage is saved in a state variable. So, we use a function to set the initial value in the state variable. Try to retrieve the value from sessionStorage. If it exists, save that. If it doesn't exist then use the intial value sent to the function.

We also inculde a useEffect hook that will run any time the key or state variable change. IF the new value in the state variable is null or undefined or an empty string then we remove the entry from sessionStorage. If the new value is not null, undefined, or an empty string then we overwrite the value in sessionStorage.

import { useState, useEffect } from 'react';

function useSessionStorage(key, initialState) {
  const [state, setState] = useState(() => {
    const storedValue = sessionStorage.getItem(key);
    return storedValue ? storedValue : initialState;
  });

  useEffect(() => {
    if (!state) {
      sessionStorage.removeItem(key);
    } else {
      sessionStorage.setItem(key, state);
    }
  }, [key, state]);

  return [state, setState];
}

export { useSessionStorage };

# Checking Authentication

In our app we are determining whether or not a user is logged in through the existence of a specific key-value pair in sessionStorage.

To do that check we can create a function that generates a component that you can put into the JSX on any component that needs to be accessed only when logged in.

// CheckAuth.jsx
import { useToken } from './context/TokenContext';
import { useEffect, useNavigate } from 'react';

export default function CheckAuth() {
  const navigate = useNavigate();
  const [token, setToken] = useToken();

  useEffect(() => {
    if (!token) {
      setToken(null); //clear the token just in case.
      navigate('/'); //send the user back to login screen
    }
  }, []);

  return null; //this is what gets placed in the JSX
}
